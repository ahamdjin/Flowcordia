# Workflow index engineering discipline

- Prefer a blocked honest state over inferred success.
- Preserve the last complete catalog until a replacement is fully proven.
- Keep exact commit identity across every GitHub read.
- Require server-owned scope and database constraints.
- Treat retries, duplicates, stale workers, and crashes as normal.
- Keep one feature owner per connection.
- Make rollback disable behavior before deleting evidence.
- Do not cross into execution infrastructure for UI convenience.
