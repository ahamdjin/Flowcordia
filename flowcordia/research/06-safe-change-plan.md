# Safe change plan

Use this order for Flowcordia changes.

## Phase 1

Map the repo in docs only.

## Phase 2

Add setup docs and grouped examples.

## Phase 3

Add a direct route that is not linked in the menu.

Visit it manually in the browser.

## Phase 4

Add a read only status page.

Show present or missing state only.

## Phase 5

Add small test actions one at a time.

Examples:

- send test email
- test GitHub connection
- test storage connection

## Phase 6

Add the navigation link after the page is stable.

## Phase 7

Add the real setup wizard.

## Checklist before any change

- route found
- service found
- data model found
- feature flag checked
- test plan written
- revert path clear
