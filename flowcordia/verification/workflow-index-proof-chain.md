# Workflow index proof chain

1. Dashboard authorization proves the signed-in user's organization/project access.
2. Server scope resolution proves the active GitHub installation, connected repository, and tracked production branch.
3. Signed push verification or a write-authorized command creates durable synchronization intent.
4. A generation- and lease-bound claim owns the attempt.
5. GitHub resolves one immutable commit.
6. Discovery proves a complete bounded tree result.
7. Exact reads prove each workflow path and blob at that commit.
8. Canonical validation and digesting prove document structure.
9. One transaction replaces the complete durable snapshot.
10. Studio re-proves exact GitHub source identity and digest.
11. A redacted DTO renders the graph.

Breaking any link blocks replacement or rendering rather than filling the gap with inferred state.
